# English-only output

All generated text and code must be in English —
comments, commit messages, documentation, resource
names, labels, annotations — even when the
conversation happens in another language.

**Why:** the project may be shared with
contributors who do not speak the user's primary
language; English keeps the codebase accessible
and consistent.

**How to apply:** when the user converses in
another language (e.g. French), still produce
English content in every file, manifest, and
commit message. The user's language only applies
to chat replies.
